▸THE REST OF THE LIBRARY

    
Lane Defense is one block of several. Each is a pure function — data in, self-contained SVG out — that runs identically in your browser (these previews), in GitHub Actions, and in Node. Sample data shown; see the repo usage docs to feed them your own.

    

  


  

    
▸HOW IT WORKS

    

      

        
Simulated, then baked

        
The tower-defense sim runs to completion at generation time. Every laser, kill, and leak becomes a CSS keyframe — the SVG that ships contains no logic, just choreography.

      

      

        
Deterministic by design

        
Same grid + level + seed → the identical battle, every render, both themes. That's what makes daily rotation honest: the cron changes the seed, not dice at view time.

      

      

        
Balanced for any profile

        
Level-3+ commit days garrison the graph. Sparse profiles promote level-2 days, dense ones get thinned, and enemy HP scales with tower count — PATROL sweeps, SIEGE leaks a few, OVERRUN ends badly.

      

      

        
Nothing to host

        
The workflow runs in your repo on GitHub's runners and commits the SVG to your own output branch. This page is static files on GitHub Pages; live previews fetch grids from the community github-contributions-api.
